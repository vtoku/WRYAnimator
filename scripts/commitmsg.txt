Fix FBX footer code so MotionBuilder loads the animation (not just skeleton)

MoBu imported a T-pose skeleton but dropped the animation and warned "legacy
file" — the FBX SDK legacy/compatibility mode. Root cause: the binary footer
code must be a computed encryption of the creation timestamp (per
hamish-milne/FbxWriter), not a generic constant; the SDK validates it and
falls back to legacy when it mismatches, which loads Models but skips the
AnimationStack graph.

- fbxBinary.ts: compute footer code = Encrypt(sourceId, mangledTime, key,
  mangledTime) from a fixed timestamp; header FileId = sourceId; FBX 7500.
- animationFbx.ts: FileId = SOURCE_ID, CreationTimeStamp = FBX_TIMESTAMP.
