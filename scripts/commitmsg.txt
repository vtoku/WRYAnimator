v0.6.0: viewport transport + trim, mocap cleaning, modern FBX for MotionBuilder

MotionBuilder import fixes (was: legacy warning, dots, no take):
- Emit modern binary FBX 7700 with a binary FileId + CreationTime, SceneInfo,
  and the Document's ActiveAnimStackName (so MoBu sets the current take).
- Remove the legacy "Takes" section (deprecated since FBX 7.4 — its presence
  made MoBu treat the file as legacy). The AnimationStack is the take; it now
  carries Version + LocalStart/Stop + ReferenceStart/Stop. AnimationLayer gets
  a Properties70 (Weight/BlendMode/...). NodeAttributes get Color/Size/LimbLength.
- Add a "Reference" Null root above the Hips so the Hips isn't the skeleton root.

Editing:
- Transport bar overlaid on the 3D viewport: play/pause, click-to-seek timeline
  with draggable in/out trim handles; trim flows end-to-end into the export
  (resample() honors trimStart/trimEnd, rebased to t=0).
- Mocap cleaning (convert/clean.ts): despike pops/hand-flips + zero-phase
  Butterworth smoothing, with UI sliders; applied to preview and export.
- Fix viewport overflowing the window (panel now scrolls internally; overlay
  stays visible).

Verified: FBXLoader round-trip (55 bones, take, 56+ tracks), trimmed export
(134s -> 67s), browser drive with cleaning + face, no console errors.
